### 研究问题一：记忆的定义与分类（第一维度，What is memory）

  - 方案A（五类，kailun）：
  显式记忆
  - 外挂纯文本非参数文本记忆，RAG/可CRUD长期记忆；
  - 多组件/多层级结构如core/semantic/episodic、图结构、记忆胶囊；
  - 固定长度压缩记忆，即固定上下文窗口长度
  隐式记忆
  - 推理时记忆，如MemGen的Memory Trigger/Viewer；
  - 参数化/模块化记忆，权重中编码）。
  
  - 方案B（四类，zhibo）：
  - 显式记忆只有外挂记忆与 context window 的区别，将外挂记忆与多层级记忆合并为一类；
  - 隐式记忆分为与大模型相关（权重或attach）与独立于大模型的可插拔模型（Memory R1/Memory Manager）。
  
  - 共识：显式/隐式总分法一致，分类可随阅读动态调整；需可落表、可操作的判定标准，明确“推理时记忆”与“参数化记忆”、以及“与大模型相关/无关”的判断规则与实例。

**初步共识方案**：

显式记忆
  - 外挂纯文本非参数文本记忆，RAG/可CRUD长期记忆；
  - 多组件/多层级结构如core/semantic/episodic、图结构、记忆胶囊；
  - 固定长度压缩记忆，即固定上下文窗口长度
  隐式记忆
  - 推理时记忆，如MemGen的Memory Trigger/Viewer；
  - 参数化/模块化记忆，权重中编码）。
  - 独立于大模型的可插拔模型（Memory R1/Memory Manager）。

### 研究问题二：优化对象是什么（第二维度，Which component is optimized）

四类对象：
  - 独立模型（pluggable/auxiliary）
  - 与大模型相关的载体（权重或attach模块）
  - Retriever（记忆/文档检索）
  - Tool probability（工具触发/选择概率策略，如JitRL）
新论文可并入既有类，否则增设新类，动态扩展体系。
  
### 研究问题三：优化目标是什么（第三维度，Target of optimization）

当前三类优化目标：
  - 回答准确度（accuracy）、
  - 效率（token/agent loop efficiency/KV cache命中率等）、
  - 行为/行动奖励（Tool action reward，基于LLM as judge对步骤“帮助度”打分）。

### 可能新增维度

  - 第四维度（价值函数设计/训练信号，What is the value function?）：定义reward到优化目标/价值函数的映射与计算流程，待明确分类准则与评估方法。
  - 第五维度（memory相关操作的细化，偏推理阶段，How memory related actions are conducted?）：若为外部存储/检索，明确谁读/写/存与策略；若依赖context window，定义读写与使用策略；若为参数化记忆，关注可编辑性/失真等挑战。边界与训练/推理环节划分需进一步明确。
  - 其他候选维度：评测维度（任务类型、数据分布、记忆长度、时间衰减）、可解释性/可编辑性、效率与成本（延迟、token、计算）。

## 补充研究方向 - 非QA训练范式

RQ1：QA reward 是否是 memory 的充分训练信号？
RQ2：QA benchmark 是否是 memory 的充分评估标准？
RQ3：能否构造非 QA 驱动的 memory 训练数据与信号？

  - 非QA向目标与基准：将对话自然度、信息压缩效率、记忆引用质量等纳入；现有许多工作最终仍以QA准确率验证，缺少直接压缩/引用质量的度量。
  - 初步结论：QA reward与QA benchmark均非充分；需系统化收集非QA训练信号与评估指标（如内部分布目标RLIF、LM自评/外部judge评分、系统读写/利用率等）。
  - 可能能跟第四个维度合并起来
